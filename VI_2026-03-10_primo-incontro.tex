\section{Discussione e decisioni}
\subsection{Prima presentazione dei componenti del gruppo}
Essendo questa la prima riunione, ogni componente del gruppo si presenta agli altri. 

\subsection{Visione collettiva dei capitolati ed espressione preferenze}
Il gruppo prende visione collettivamente dei capitolati e, ad uno ad uno, viene espressa una preferenza. Il gruppo, dopo varie discussioni, decide di dedicarsi ai capitolati (in ordine di preferenza):
\begin{enumerate}
    \item \textbf{C6}: Second Brain - Zucchetti S.p.A.
    \item \textbf{C5}: NEXUM - Eggon
    \item \textbf{C2}: Code Guardian - Var Group S.p.A.
\end{enumerate}
Si rimanda la decisione sul capitolato \textbf{C4} e si decide di contattare l'azienda per delucidazioni sul progetto.

\subsection{Creazione indirizzo di posta elettronica}
Il gruppo crea l'indidirizzo di posta elettronica \textit{thebitless.swe@gmail.com} con cui contattare le aziende e i docenti.

\subsection{Stesura mail per richiesta colloquio alle aziende}
Vengono redatte e spedite le email per richiesta di colloquio alle aziende: Zucchetti (per capitolato C6), Miriade (per capitolato C4) ed Eggon (per capitolato C5).
In vista degli incontri con le aziende, ogni membro del gruppo prepara delle domande sul capitolato.

\subsection{Prima definizione di \textit{Way of Working}}
Il gruppo comincia la discussione del \textit{Way of Working}, ed in particolare:
\begin{itemize}
    \item Scelta dei tool per versionamento e gestione del progetto
    \item Bozza delle linee guida sulla gestione delle repository e del naming
\end{itemize}
Per il primo punto si è scelto di usare Git (per il versionamento) e GitHub (per la creazione e condivisione delle repository). Per la gestione del progetto si è deciso di affidarsi a Github Projects che offre un ottimo sistema Kanban integrato con le repository e le issues.
A questo proposito si è creata l'organizzazione \textbf{The Bitless Group SWE} su GitHub in cui tutto il team è stato inserito.

Per il secondo punto invece è stata scritta una bozza di Wiki con le linee guida sulla corretta gestione delle repository e del naming dei file.
